# Пошаговая инструкция по настройке GitHub Pages

## Шаг 1: Удалите неправильный файл main.yml

Если в GitHub есть файл `.github/workflows/main.yml` с ошибкой, удалите его:
1. Перейдите в репозиторий на GitHub
2. Откройте `.github/workflows/main.yml`
3. Нажмите на иконку корзины (Delete)
4. Подтвердите удаление

## Шаг 2: Запушите правильный файл deploy.yml

Выполните эти команды в терминале:

```bash
git add .
git commit -m "Fix workflow configuration"
git push origin main
```

## Шаг 3: Включите GitHub Pages

1. Перейдите в ваш репозиторий на GitHub
2. Нажмите **Settings** (Настройки)
3. В левом меню найдите **Pages**
4. В разделе **Source** выберите:
   - **Source**: `GitHub Actions` (не "Deploy from a branch")
5. Сохраните изменения

## Шаг 4: Запустите workflow вручную (если нужно)

1. Перейдите на вкладку **Actions**
2. Выберите workflow **"Build and Deploy"**
3. Если есть кнопка **"Run workflow"**, нажмите её
4. Дождитесь завершения (зелёная галочка)

## Шаг 5: Проверьте результат

После успешного деплоя (1-2 минуты):
- URL будет: `https://snezhnaya55555-rgb.github.io/obyavleniya/`
- Или: `https://snezhnaya55555-rgb.github.io/` (если это user site)

## Если что-то не работает:

1. Проверьте вкладку **Actions** - там должны быть логи ошибок
2. Убедитесь, что в Settings → Pages выбран **"GitHub Actions"**, а не ветка
3. Проверьте, что файл `.github/workflows/deploy.yml` существует и правильный

## Установка PWA

После успешного деплоя:
1. Откройте сайт в браузере
2. В меню браузера (три точки) выберите "Установить приложение"
3. Или нажмите на иконку установки в адресной строке
